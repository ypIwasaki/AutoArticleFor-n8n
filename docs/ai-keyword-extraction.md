# AI Keyword Extraction Instructions

The workflow does not call OpenAI or any other AI API automatically.

Instead, each workflow run writes a prompt/instruction Markdown file that can be
copied into an AI chat manually:

```text
content/ai-extraction-instructions/YYYY-MM-DD.md
```

That file contains:

- The current search keywords.
- The extraction request for semantic keyword candidates.
- Inclusion and exclusion criteria.
- A requested Markdown output format.
- Rule-based candidates as reference data.
- The full saved daily digest as input context.

The instruction tells the AI to save semantic extraction results as a Markdown
file in the project when file access is available:

```text
content/ai-keyword-candidates/YYYY-MM-DD.md
```

The source digest and rule-based candidate files are saved separately:

```text
content/daily-digests/YYYY-MM-DD.md
content/keyword-candidates/YYYY-MM-DD.md
```

`content/keyword-candidates-ai/` is legacy output from an earlier workflow
version and is not generated by the current workflow.
